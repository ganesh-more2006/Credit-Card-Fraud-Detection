# 🛡️ Credit Card Fraud & Risk Analytics Dashboard

### 📌 Project Overview
This project is an end-to-end data analytics solution designed to identify high-risk loan applicants. By analyzing a dataset of over 300,000 records, the project provides actionable business insights through an interactive web-based dashboard. 

The goal is to help financial institutions minimize losses by detecting potential defaulters based on their demographic and financial profiles.

---

### 🚀 Key Features
- **Exploratory Data Analysis (EDA):** Comprehensive analysis of 122 features to find correlations with loan defaults.
- **SQL-Powered Backend:** Data is managed and queried using a structured SQLite database for high performance.
- **Interactive Executive Dashboard:** Built with Streamlit, allowing users to filter data by income, loan type, and more.
- **Client Search Engine:** A feature to instantly check the risk status of any specific client using their ID.
- **Advanced Visualizations:** Includes Sunburst charts, Scatter plots, and Trend lines to visualize complex risk factors.

---

### 🛠️ Tech Stack
- **Languages:** Python (Pandas, Plotly, Seaborn)
- **Database:** SQL (SQLite3)
- **Framework:** Streamlit (For Web Application)
- **Tools:** Jupyter Notebook, VS Code, Git

---

### 📊 Major Business Insights
1. **Age-Based Risk:** Applicants in the 20-30 age group show the highest default rate, suggesting a need for stricter verification for younger clients.
2. **Credit Scoring:** External source scores (EXT_SOURCE_2) are the strongest predictors of fraud/default.
3. **Education Impact:** Higher education levels correlate with lower default rates across all genders.

---

### 📂 Project Structure
- `app.py`: The main Streamlit application script.
- `fraud_data.db`: SQLite database containing cleaned loan data.
- `analysis.ipynb`: Jupyter Notebook detailing the data cleaning and analysis process.
- `requirements.txt`: List of dependencies for cloud deployment.
